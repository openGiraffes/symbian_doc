List browsing

Lists can be browsed in two directions. Pressing the Arrow up key moves the focus one step up (backward), and pressing the Arrow down key moves the focus one step down (forward).

When browsing with hardware keys, the item in focus must always be visible. The detailed rules of moving the focus are as follows:

  • If the item about to be chosen is already fully visible, all items remain in their current positions, and the highlight simply moves from the old item to the new one.

  • If the new item that the user wishes to choose is not visible, all items are moved in the view in the appropriate direction so that the new item becomes fully visible. For example, if the focus is moving down and the new item is currently below the bottom edge of the view, the items are moved up.

  • When items in the view are moved, it is only for the minimum amount necessary. For example, when moving items up to get the next one under the bottom edge visible, the item moves to the lowest allowed position in the view.

By default, all lists loop when moving with navigation keys. This means it is possible to browse forward from the last choice item, because it leads to the first item in the list, and vice versa.

It is possible to specify that a list is a queue. This means that it is not allowed to browse forward from the last item or backward from the first item. If the user attempts this, the list does not react; there is no feedback except for the normal keypad tone.
