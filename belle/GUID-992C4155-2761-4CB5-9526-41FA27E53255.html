Category: CdlEng

Panic of client raised by the Cdl Engine.

0

This panic occurs when there is no instance of CCdlEngine.

1

The panic occurs when the stack ID of CCdlCustomisationStack is not present.

2

This panic occurs when the sublayer ID and the reference ID are not compatible.

3

This panic occurs when the CCdlInstance cannot be implemented.

4

This panic occurs when merging of known and unknown id is not possible.

5

This panic occurs when the reference index is out of range.

6

This panic occurs when there is a bad reference count.

7

This panic occurs when there is no instance loaded for the CCdlCustomisationStack.

8

This panic occurs when the interface of CCdlInstanceProxy is not intialised.

9

This panic occurs when the library manager is already set.

10

This panic occurs when the CCdlChangeMonitor tries to change the reference type.

11

This panic occurs when the delete assertion fails.

12

This panic occurs when the reference ID is not set.

13

This panic occurs when the library is found. This occurs in debug mode only.
