Column structure of lists

For visual consistency, the standard list layouts are built around a structure of three virtual columns. The column borders are aligned with the sides of the context pane.

The list items can be divided into three sections (columns A, B and C). All three columns need not be used separately in a list layout: combinations AB, BC or ABC are possible. Column D is basically the very end of column C where additional indicators can be displayed. The area for these indicators is not strictly an individual column in the same sense as the other three, because it can be used dynamically, item by item. All items within a list must use the same column layout.

Figure: Different column arrangements: a) All columns used for a text item. b) Column A contains an icon, columns B&C combined for text. c) Columns A&B combined for a large icon, column C contains text. d) Column A: an icon; column B: a label; column C: text.

Table: Standard elements within list item associated with columns
Column Description

A

  • Small graphic (icon): an item property indication.

  • The item number (see Numbered items).

B

Heading (the title or attribute of the item).

AB

  • Heading (the title or attribute of the item).

  • Large graphic (for example, an icon or an image thumbnail).

C/BC/ABC

The main text of the item.

D

Icon or marking mode mark.

Note: The list layout can be mirrored for localizations such as Arabic and Hebrew (for example, column A is on the right edge of the pane). See Layout changes in bi-directional languages.
