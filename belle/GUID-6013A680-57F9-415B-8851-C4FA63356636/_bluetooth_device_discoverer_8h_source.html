00027 Description:  CBluetoothDeviceDiscoverer is responsible for displaying to the player (Master role) the available
00028                 BT devices in range from which they can choose. CBluetoothDeviceDiscoverer is implemented using the
00029                 Rnotifier dialog that is provided by the Notifier Framework. If the application needs to customize the 
00030                 BT discovery [by displaying to the user the BT devices in range using a custumize UI] then the
00031                 application needs to use the RHostResolver class instead of the RNotifier class. Steps to use
00032                 the RHostResolver are:
00033  
00034                 1) RHostResolver::Open(RSocketServ &aSocketServer, TUint anAddrFamily, TUint aProtocol);
00035                 2) RHostResolver::GetByAddress(const TSockAddr &anAddr, TNameEntry &aResult, TRequestStatus &aStatus);
00036                 3) In the RunL store the BT address and call RHostResolver::Next() to get the next availble BT device.
00037                 4) When the BT discovery is finished then pass the array of BT device addresses to the engine (by a callbck)
00038                    to be displayed by the customized UI (like a multi choice control).
00039                 5) The Master player selects the BT devices to which she wants to connect.
00040                 6)The App engine will create one Connector per BT device selected by the Master player
00041  
00042 */
