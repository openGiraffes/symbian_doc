Configuring Comms

This section contains conceptual information required to configure the Comms Infrastructure.

  • What is the Comms Database?
    The Comms Database contains the information necessary to configure components in the comms stack.
  • What is the System Critical Parameter?
    System Critical Parameters allow the OS to reset when a module crashes. A System Critical Parameter compirses 2 distinct paramters.
  • What are Multi-Threaded Servers?
    A multi-threaded server is a server that has more than one thread. This topic describes the root server which comprises 2 multi-threaded servers.
  • What are ESK files?
    ESK files contain protocol module information that is used to load ESock.
  • What is IP Security Policy (IPSecPol)?
    This topic describes the IPSecPol.
  • What is Serial Port Contention?
    Serial Port Contention occurs when ports are not available to open client applications. This topic outlines problems and suggests solutions that Serial Port Contention raises with the C32 Serial Server Client.
