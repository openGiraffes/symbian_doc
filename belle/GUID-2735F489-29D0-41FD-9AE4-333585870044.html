Category: AVKON SKINS

The panic category for Avkon skins and these occur in debug mode only.

—6000

The panic occurs if the color RGB is not indexed in the CAknsColorTableItemData.

-6001

The panic occurs when there is a decrease of reference count in the CAknsAppSkinInstanceCacheEntry.

—6002

The panic occurs when the release item is under flow defined in the CAknsAppSkinInstance.

—6003

The panic occurs when the local item of CAknsAppSkinInstance has a invalid parameter.

—6004

The panic occurs when the windows size is not compatible.

—6005

The panic occurs when the position for the drawable window is not present.

—6006

The panic occurs when the windows had a zero lay dimension.

—6007

The panic occurs when there is a bad alignment of the parent pointer location.

—6008

The panic occurs when the write operation is in a bad state.

—6009

The panics due to incorrect usage of the Avkon skin.
