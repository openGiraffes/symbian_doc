Dimming

Dimming the background

The entire screen is dimmed in the background when the Options and Power menu, notes, soft notifications, settings editors or queries are displayed. Discreet pop-ups are an exception.

Dimming controls

In some situations, it may be necessary to dim unavailable controls. This is done on the component level so that a control has a normal and a dimmed state, for example, toolbar button.

Dimming indicates that the control is temporarily unavailable. For example, function is not possible for the focused item, but for the other items in the view. Dimming should not be used if the user never sees the control in an active state within that view; in such cases, the unavailable control should be hidden.
