Data queries with multiple fields

It is possible to have two input fields in a Data query. An example of this is a user name and password query: one field is a normal alphanumeric editor and the other a password editor. In this case, a press of the Selection key moves the insertion point from the first field to the next; in the second field it accepts the query. The left softkey always accepts the query.

The user can also move from one field to the other using the Arrow up and Arrow down keys or by tapping. Note that in landscape orientation it may not be feasible to use a query with multiple fields and alternative solutions may be necessary (for example, launching one query first for user name and on pressing OK another query is launched for the password).
