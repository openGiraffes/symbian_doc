Porting applications from S60 5th Edition to Symbian^3

This section describes the changes required for applications to work on Symbian^3.

Steps

  • Add the following macro in your application's MMP files. This is required because the structure of the source code has changed.

    APP_LAYER_SYSTEMINCLUDE

  • Modify the applications to support single-tap.

  • Modifying applications to support single-tap
    Symbian^3 enhances the touch interaction by providing support for single-tap.
  • Updating the platform UID
    This topic describes how to update the platform UID.
  • API Changes and Compatibility Breaks
    The purpose of this section is to highlight all the API changes and compatibility breaks (binary & source) in Symbian^3. It provides all the required information for Symbian application developers and eases their migration process from S60 5th Edition to Symbian^3.
  • DSA Migration Guide
    This migration guide explains the guidelines that applications that use direct screen access (DSA) must follow in order to run on ScreenPlay (NGA) devices.
