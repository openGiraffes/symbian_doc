On-Target Debugging

On-target debugging refers to debugging environments where code is run on device hardware. This increases the debugging challenge due to the restrictive nature of such hardware.

When debugging Comms software in such environments, the following tools are available to help:

  • D_Exe

  • TRK

  • Lauterbach

  • Crash Analyst

  • dtrace

The RDebug::Print() facility is rarely used by default by Symbian platform Comms software for debug tracing since the Comms software mostly uses CDU. If required, CDU's output can be redirected to the same output port used by the RDebug interface.
