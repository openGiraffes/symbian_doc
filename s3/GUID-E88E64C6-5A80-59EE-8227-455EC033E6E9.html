What are sessions with the Comms Database

This topic describes the use of sessions to access the Comms Database.

Tools and applications create and use a session to access the Comms Database. A session is an instance of the class [[[ERROR: [NOKX000E] Unable to find definition for key reference 'CommsDat']]]CMDBSession. An instance of CMDBSession acts as a handle to the database. The functions provided by [[[ERROR: [NOKX000E] Unable to find definition for key reference 'CommsDat']]]MMetaDatabase to access the Comms Database require an instance of CMDBSession.

You must create a session before you get data from the Comms Database and before you update the Comms Database. To create a session, you create an instance of the [[[ERROR: [NOKX000E] Unable to find definition for key reference 'CommsDat']]]CMDBSession class.

You delete the session after you finish your use of the Comms Database. To delete a session, you delete the instance of the [[[ERROR: [NOKX000E] Unable to find definition for key reference 'CommsDat']]]CMDBSession class.
